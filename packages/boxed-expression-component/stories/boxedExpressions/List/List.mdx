# Boxed List

A boxed list enables you to create a list of boxed expressions.

## Overview

### Index column

#### Header

The header row has a "#" symbol which is a placeholder and has any purposes.

#### Rows

The first and any other additional rows has the row number.

### Items columns

#### Header

The header of the Boxed List expression will only exist if it's the root expression of a Decision node,
and the header will represets the Decision name and type. Clicking on it a context menu will open enabling to edit its values.

#### Items

The items can be any type of expression, even other boxed list.

### Controls

##### Resizing

Hovering the mouse on the header or item cell will show a resize handle on the right side of the cell.
Clicking on it and draging enables to resize the cell to the minimum of 190px and without a maximum size.

**Important**: This action is only possible if an item has an expression, and this operation will resize all cells in the column.

#### Items

Left clicking on the index column cells or in the header cell (if applicable) will open the "Row" context menu.
It's possible to insert above, insert below, insert multiples and delete the row.

Another way to add columns is by hovering the mouse on the index column cells will make appear a plus sign on the top (insert above)
or on the bottom (insert below) of the cell, the position of the plus sign will depend of the mouse position.
Right clicking on it will add a new row above or below of the actual row.

**Important**: It's not possible to perform the above process in the header row index ("#").
